# 簇内精填修正报告

## 📋 概述

本报告详细记录了基于聚类结果的簇内精细化缺失值填充过程，通过按簇分别处理，实现了比全局粗填更精准的缺失值修正。

---

## 🎯 精填目标

在全局KNN粗填的基础上，利用聚类结果进行簇内精细化填充，进一步提升数据质量：
- **数值型特征**：使用簇内中位数或簇内KNN
- **类别型特征**：使用簇内众数
- **兜底机制**：避免在样本不足时强行精填

---

### 3️⃣ **数值型特征精填策略**

虽然本次数据中主要是类别型特征缺失，但脚本已准备好数值型特征的精填策略：

#### 策略选择逻辑

```python
if 缺失率 < 10%:
    使用簇内中位数填充  # 稳健，适配偏态分布
else:
    使用簇内KNN精填     # 缺失率高时用更精细的方法
```

#### 为什么用中位数而非均值？

| 指标 | 中位数 | 均值 |
|------|--------|------|
| **稳健性** | ✅ 高（不受极端值影响） | ❌ 低（受极端值影响大） |
| **偏态分布** | ✅ 适配 | ❌ 不适用 |
| **二手车价格** | ✅ 右偏分布友好 | ❌ 会偏向高价 |

**示例**：
- 某簇车龄中位数：12年
- 某簇车龄均值：15年（被少量老车拉高）
- → 使用中位数更能代表该簇的典型情况

#### 簇内KNN精填

当缺失率 > 10% 时，采用簇内KNN：

1. **选择相关特征**：计算与目标特征的相关性，选取Top 10
2. **簇内KNN**：只在当前簇内进行KNN填补（k=5）
3. **优势**：利用簇内相似样本，更精准

---

### 4️⃣ **类别型特征精填策略**

#### 主要策略：簇内众数填充

```python
cluster_mode = train_cluster[col].mode()
填充值 = cluster_mode[0]  # 出现频率最高的类别
```

**示例**：
- 簇0的`bodyType`众数：轿车
- 簇1的`fuelType`众数：汽油
- 簇2的`gearbox`众数：手动挡

#### 备选策略：未知类别

当簇内缺失率 > 50% 时：
```python
填充值 = 'Unknown'  # 保留为未知类别
```

**原因**：
- 缺失过多，众数不可靠
- 保留"未知"作为新类别，让模型学习

---

### 5️⃣ **兜底机制**

#### 防止过度精填

```python
if 簇内完整样本数 < k_neighbors:
    保留粗填结果  # 不强行精填
```

**参数**：
- `k_neighbors = 5`

**场景示例**：
- 簇2只有4,506样本
- 某特征在该簇只有3个完整样本
- → 3 < 5，保留全局KNN粗填结果

**价值**：
- ✅ 避免在小样本上强行KNN导致误差放大
- ✅ 保证精填不会比粗填更差
- ✅ 稳健优先于激进

## 💡 精填 vs 粗填对比

### 全局粗填（第一步）

**方法**：全局KNN (k=7)

**优点**：
- ✅ 快速高效
- ✅ 利用全部数据的相关性
- ✅ 适合初步填补

**缺点**：
- ❌ 忽略群体差异
- ❌ 可能跨簇混合不同分布
- ❌ 不够精准

### 簇内精填（第三步）

**方法**：按簇分别中位数/KNN

**优点**：
- ✅ 考虑群体特征
- ✅ 避免跨簇污染
- ✅ 更精准贴合分布
- ✅ 中位数适配偏态分布

**缺点**：
- ❌ 计算量较大（需遍历每个簇）
- ❌ 小簇可能样本不足

### 综合策略

```
全局KNN粗填 (k=7)
    ↓
K-Means聚类 (k=5)
    ↓
簇内精填（中位数/众数/小范围KNN）
    ↓
最终高质量数据集
```

## 🎯 关键技术要点

### ✅ 正确做法

1. **按簇分别处理**
   - 每个簇独立计算统计量
   - 避免跨簇信息泄露

2. **中位数优先**
   - 适配二手车特征的偏态分布
   - 比均值更稳健

3. **兜底机制**
   - 样本不足时保留粗填结果
   - 避免引入更大误差

4. **缺失率阈值**
   - <10%：用中位数/众数
   - >10%：用KNN精填
   - >50%：用"Unknown"

### ❌ 常见错误

1. ❌ 直接用全局均值/中位数
2. ❌ 不考虑簇的差异
3. ❌ 小样本强行KNN
4. ❌ 没有兜底机制
