#### 1.2 粗填（Coarse Imputation）— 全局KNN缺失值填充

##### 方法概述

粗填是整个数据预处理流程中的**第一轮全局缺失值填充**，在所有特征工程完成之后、聚类分析之前执行。其目的是快速、高效地为所有数值型特征提供合理的初始填充值。

##### 核心方法：KNNImputer

```python
from sklearn.impute import KNNImputer
imputer = KNNImputer(n_neighbors=7, weights='distance')

# 在训练集上fit_transform，在测试集上只transform（防止数据泄露）
train_filled = imputer.fit_transform(train_numerical)
test_filled = imputer.transform(test_numerical)
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `n_neighbors` | **7** | 参考7个最近邻样本的值进行加权估计 |
| `weights` | **'distance'** | 距离越近的邻居权重越大，提供更精确的填补 |

##### 为什么选择KNN而非均值/中位数？

| 对比维度 | KNN填充 ✅ | 全局均值/中位数 ❌ |
|---------|-----------|-------------------|
| **分布保持** | 保留特征的分布特性和方差 | 会缩小方差，破坏分布 |
| **特征相关性** | 利用特征间的相关性，多特征协同填补 | 完全忽略特征间关系 |
| **样本相似度** | 基于相似样本填补，更具有物理意义 | 对所有样本"一刀切" |
| **后续聚类** | 填补值合理，有利于聚类发现真实结构 | 引入偏差，干扰聚类质量 |

**核心原则**：在二手车数据中，不同品牌的车辆功率、里程差异巨大。一辆保时捷的缺失功率值不应参考全数据集的均值，而应参考与它品牌、车龄、价格相似的其他车辆——这正是KNN的核心思想。

##### 粗填的定位：为聚类提供基础

```
原始数据（存在缺失值）
    ↓
特征工程（创建衍生特征）
    ↓
【粗填】KNNImputer (k=7, distance-weighted)  ← 当前步骤
    ↓
特征标准化（Z-score）
    ↓
K-Means 聚类（基于完整数据）
    ↓
【精填】簇内精细化修正  ← 下一步
```

粗填的核心价值在于：**确保聚类分析有完整的数据输入**。K-Means算法无法处理缺失值，必须在聚类前完成填充。但全局KNN填充存在跨群体"污染"的问题——不同簇的样本被混在一起计算近邻，可能导致填充值不够精准。这正是后续精填要解决的问题。

#### 3.2 最优簇数确定 — 手肘法 + 轮廓系数

确定K-Means的最优簇数k是聚类分析中最关键的决策之一。本项目采用**手肘法（Elbow Method）**和**轮廓系数（Silhouette Score）**两种互补方法联合判断。

---

##### 方法一：手肘法（Elbow Method）

**原理**：随着簇数k增加，SSE（Sum of Squared Errors，簇内误差平方和）必然单调递减。但当k超过"真实"簇数后，SSE的下降速度会显著变缓，形成类似手肘的拐点。

```
SSE = Σᵢ Σₓ∈Cᵢ ||x - μᵢ||²

其中：
  Cᵢ = 第i个簇
  μᵢ = 簇Cᵢ的中心点
  x  = 簇内的样本点
```

**拐点判读方法**：

```
SSE
 ↑
 |  ╲
 |    ╲
 |      ╲  ← 快速下降区（每增加一个簇都显著改善）
 |        ╲
 |          ● ← 拐点（最优k）  
 |           ╲
 |             ╲___  ← 平缓区（增加簇数改善甚微）
 |                 ╲___
 +──────────────────────────→ k
```

**手肘法的局限**：拐点有时不够明显（曲线平滑下降时），难以精确判断。因此需要轮廓系数作为补充。

---

##### 方法二：轮廓系数（Silhouette Score）

**原理**：轮廓系数综合衡量了**簇内紧密度**（样本与同簇其他样本的距离）和**簇间分离度**（样本与最近邻簇的距离）。取值范围 [-1, 1]，值越大表示聚类质量越好。

**计算公式**：

```
对于单个样本i：
  a(i) = 样本i到同簇内其他所有样本的平均距离（簇内紧密度）
  b(i) = 样本i到最近的其他簇中所有样本的平均距离（簇间分离度）
  
  轮廓系数 s(i) = (b(i) - a(i)) / max(a(i), b(i))

整体轮廓系数 = 所有样本s(i)的均值
```

**直观理解**：

```
s(i) ≈ +1  →  a(i) ≪ b(i)  →  ✅ 样本离同簇样本很近，离其他簇很远（理想状态）
s(i) ≈  0  →  a(i) ≈ b(i)  →  ⚠️ 样本处于两个簇的边界上（模糊状态）
s(i) ≈ -1  →  a(i) ≫ b(i)  →  ❌ 样本被错误分配，离其他簇比离自己簇更近（糟糕）
```

---

##### 轮廓系数判定标准

由于本项目使用38维高维特征进行聚类，轮廓系数的绝对值会低于低维场景（维度灾难效应）。以下是结合文献和本项目实际情况的判定阈值：

| 轮廓系数 | 评级 | 含义 | 处理建议 |
|---------|------|------|---------|
| **0.50 ~ 1.00** | ⭐⭐⭐ 优秀 | 簇结构清晰，簇内紧密、簇间分离 | 可直接使用聚类结果 |
| **0.25 ~ 0.50** | ⭐⭐ 良好 | 簇结构可接受，存在一定重叠 | 可以接受，高维数据常见此范围 |
| **0.10 ~ 0.25** | ⭐ 一般 | 簇结构较弱，边界模糊 | 检查特征选择、尝试其他k值 |
| **< 0.10** | ❌ 差 | 簇结构非常模糊，近随机分配 | 需要重新设计聚类方案 |
| **< 0 (负值)** | ❌❌ 严重问题 | 样本被错误分配 | 聚类失败，检查数据和算法 |

**维度对轮廓系数的影响**：

```
为什么高维数据的轮廓系数偏低？

在38维空间中：
  - 任意两点间的欧氏距离趋于相近（维度灾难）
  - a(i)和b(i)的差异被缩小
  - 导致 s(i) = (b-a) / max(a,b) 趋向于较小的正值
  
因此在高维场景下：
  - 轮廓系数 0.10 ~ 0.20 → 属于正常可接受范围
  - 轮廓系数 0.25+       → 已经表示较好的簇结构
  - 不应期望低维场景 0.7+ 的水平
```

```
步骤1：手肘法扫描
  ├── 计算 k=2,3,4,...,15 的 SSE
  ├── 绘制 SSE vs k 曲线

      ↓
步骤4：最终决策
  └── 综合手肘法拐点 + 最大轮廓系数 + 业务合理性 → 确定最优k
```

#### 3.5 精填（Refined Imputation）— 簇内精细化缺失值修正

##### 方法概述

精填是在粗填+K-Means聚类之后执行的**第二轮精细化缺失值填充**。其核心思想是：**每个簇内的样本具有相似的特征分布，在簇内进行填充比全局填充更加精准**。

##### 为什么需要精填？—— 粗填的局限性

全局KNN粗填存在一个关键问题：**跨簇污染**。

```
示例：一辆法拉利（簇1-高价车）的 fuelType 缺失
    ├── KNN粗填 (k=7)：在全量150,000样本中找7个最近邻
    │   → 近邻可能来自不同的价格簇
    │   → 可能混入低价车（簇0）的fuelType偏好
    │   → 填充结果可能不准确
    │
    └── 簇内精填：只在簇1（38,313个高价车样本）内找最近邻
        → 近邻都是同价位、同级别的车辆
        → 填充值更贴合该群体的实际分布
        → ✅ 更精准
```

##### 精填策略矩阵

| 特征类型 | 缺失率 | 填充方法 | 说明 |
|---------|--------|---------|------|
| **数值型** | < 10% | 簇内中位数 | 稳健，不受极端值影响，适配偏态分布 |
| **数值型** | ≥ 10% | 簇内KNN (k=5) | 缺失率高时用更精细的方法，选取Top10相关特征 |
| **类别型** | < 50% | 簇内众数 | 使用簇内出现频率最高的类别 |
| **类别型** | ≥ 50% | 填充为 `'Unknown'` | 缺失过多时保留为未知类别，让模型自行学习 |

##### 为什么数值型用中位数而非均值？

二手车数据中的数值特征（如价格、功率、里程）普遍呈**右偏分布**（少数极端值拉高均值）。

| 场景示例 | 中位数 | 均值 | 哪个更合理？ |
|---------|--------|------|-------------|
| 簇0功率分布：大部分100-200马力，少数>500 | **150** | 185 | ✅ 中位数代表该簇的典型车辆 |
| 簇1车龄分布：大部分3-8年，少数>20年 | **5年** | 7.2年 | ✅ 中位数不受少量老车影响 |

**结论**：中位数比均值更能代表簇内"典型样本"的特征值。

##### 簇内KNN精填流程

当缺失率较高（≥10%）时，使用簇内KNN进行精填：

```
1. 筛选相关特征
   ↓ 计算该特征与簇内其他特征的相关性
   ↓ 选取 Top 10 强相关特征
   
2. 构建KNN填补器
   ↓ KNNImputer(n_neighbors=min(5, 簇内完整样本数))
   ↓ 只在当前簇内进行fit
   
3. 执行填补
   ↓ 对簇内训练集缺失值进行填充
   ↓ 对簇内测试集缺失值进行填充
   
4. 兜底检查
   ↓ 如果簇内完整样本 < k_neighbors
   ↓ → 保留粗填结果，不强行精填
```

##### 兜底机制（关键安全保障）

```python
if 簇内完整样本数 < k_neighbors:
    保留粗填结果  # 避免在小样本上强行KNN导致误差放大
```

**场景**：假设某特征在簇2（仅4,506样本）中只有3个完整值。此时 `3 < 5（k_neighbors）`，KNN近邻不足，强行填充反而会引入更大误差。因此**保留粗填结果**是更稳妥的选择。

**兜底机制的价值**：
- ✅ 确保精填不会比粗填更差
- ✅ 在小样本簇中保持数据稳定性
- ✅ 稳健优先于激进

##### 精填 vs 粗填 对比

| 维度 | 粗填（第一阶段） | 精填（第二阶段） |
|------|----------------|-----------------|
| **范围** | 全局（所有样本） | 簇内（同类样本） |
| **方法** | KNN (k=7, distance) | 中位数/众数/KNN (k=5) |
| **执行时机** | 聚类之前 | 聚类之后 |
| **优势** | 快速，利用全部数据 | 精准，避免跨簇污染 |
| **劣势** | 可能跨群体污染 | 小簇可能样本不足 |
| **兜底** | 无（依赖KNN自身） | 样本不足时保留粗填 |

##### 完整两阶段策略

```
数据（含缺失值）
    │
    ├── 步骤1：全局KNN粗填 (k=7)
    │   ├── 为所有特征提供初始填充值
    │   └── 确保聚类分析有完整数据
    │
    ├── 步骤2：K-Means聚类 (k=5)
    │   └── 将样本分为5个同质群体
    │
    └── 步骤3：簇内精填修正
        ├── 数值型 → 中位数优先 / KNN备选
        ├── 类别型 → 众数优先 / Unknown备选
        └── 兜底 → 样本不足保留粗填
            ↓
        最终高质量数据集
```

#### 3.6 最小簇的识别与合并

##### 问题背景

在实际的K-Means聚类结果中，可能出现样本量极少的簇。例如本项目K5数据的**簇4仅包含15个样本**（0.01%）。这类极小簇会带来以下问题：

| 问题 | 说明 |
|------|------|
| **过拟合风险** | 15个样本无法支撑有效的模型学习，模型会"死记"而非"理解" |
| **统计不稳定** | 极小簇内的均值、中位数等统计量不具有代表性 |
| **噪声放大** | 如果这15个样本是异常值，可能污染相邻簇的边界 |
| **精填失效** | 簇内样本太少，无法执行KNN精填（完整样本数 < k_neighbors） |

##### 最小簇的判定标准

```
极小簇判定条件（满足任一即视为极小簇）：
  1. 样本数 < 100
  2. 占比 < 1%（对于本项目150,000训练集，即<1,500样本）
  3. 样本数 < k_neighbors × 2（无法执行有效的簇内KNN）
```

本项目中，簇4仅15个样本，同时满足以上三个条件，明确判定为**极小簇**。

##### 合并策略：基于欧氏距离的最近邻合并

极小簇不应直接丢弃（可能包含有价值的边界样本），而是应合并到与其**特征最相似**的相邻簇。

**步骤**：

```
1. 计算极小簇的特征中心
   ↓ 计算该簇所有特征的标准均值向量
   
2. 计算与其他簇的欧氏距离
   ↓ d(cluster_A, cluster_B) = ||μ_A - μ_B||₂
   ↓ 其中μ_A和μ_B是标准化后的簇中心向量
   
3. 选择距离最近的簇
   ↓ 目标簇 = argmin(d(极小簇, 其他簇))
   
4. 执行合并
   ↓ 将极小簇的样本标签改为目标簇的标签
   
5. 验证合并合理性
   ↓ 检查合并后簇的样本分布和价格统计
   ↓ 确认不会引入异常值
```

##### 本项目实际案例

```
簇4特征均值（15个样本）:
  • power: 600.00
  • kilometer: 8.83
  • carAge: 11.27
  • price: 8,068.33

与其他簇的标准化欧氏距离:
  • 簇0: 7.9278
  • 簇1: 10.2907
  • 簇2: 5.7028  ← 最近 ✅
  • 簇3: 10.4482
```

**结论**：簇4与簇2的特征分布最为接近，将簇4的15个样本合并到簇2。

| 簇ID | 合并前样本数 | 合并后样本数 | 占比 |

- **最小簇处理**（详见 §3.6）：识别并合并极小簇（<100样本），减少噪声干扰
