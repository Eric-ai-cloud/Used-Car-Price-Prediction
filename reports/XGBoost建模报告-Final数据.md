## 🏗️ 二、模型配置

### 2.1 XGBoost超参数

```python
{
    'objective': 'reg:squarederror',     # 回归任务
    'eval_metric': 'rmse',               # RMSE评估
    'max_depth': 8,                      # 树最大深度
    'learning_rate': 0.05,               # 学习率
    'n_estimators': 1000,                # 树的数量
    'min_child_weight': 5,               # 最小子节点权重
    'subsample': 0.8,                    # 样本采样比例
    'colsample_bytree': 0.8,             # 特征采样比例
    'gamma': 0.1,                        # L2正则化
    'lambda': 1.0,                       # L1正则化
    'random_state': 42,                  # 随机种子
    'n_jobs': -1                         # 使用所有CPU核心
}
```

### 2.2 训练策略

- **验证集划分**: 80% 训练 / 20% 验证
- **早停机制**: 50轮无改善则停止
- **实际迭代**: K5=999轮, K6=997轮

## 📈 三、模型性能对比

### 3.1 整体性能

| 模型 | RMSE | MAE | R² | 说明 |
|------|------|-----|----|----|
| **之前模型** (原始数据) | 1131.22 | 509.74 | 0.9699 | 删除异常值 |
| **K5 Final** | 1267.30 | 530.40 | **0.9707** | Power截断+重算 |
| **K6 Final** | 1260.16 | 530.23 | **0.9710** | Power截断+重算 |

### 3.2 关键发现

#### ✅ 优势
1. **R²提升**: Final数据的R²都高于之前模型
   - K5: 0.9699 → 0.9707 (+0.08%)
   - K6: 0.9699 → 0.9710 (+0.11%)

2. **数据完整性**: 保留全部150,000条样本（未删除）

3. **符合规则**: Power严格在[0, 600]范围内

#### ⚠️ 劣势
1. **RMSE上升**: 1131 → 1260/1267 (+11-12%)
   - 原因: 截断保留了部分噪声样本
   - 之前删除了3,365条异常记录

2. **MAE上升**: 509 → 530 (+4%)
   - 与RMSE类似，整体误差略有增加

## 💡 五、关键发现总结

### 5.1 Final数据的价值

✅ **R²提升**: 0.9699 → 0.9710  
✅ **衍生特征价值凸显**: power_age_ratio进入Top3  
✅ **数据完整性**: 保留全部样本  
✅ **符合比赛规则**: Power在[0, 600]范围  

### 5.2 K5 vs K6 的选择

#### 🏆 推荐K6的理由
- R²最高 (0.9710)
- RMSE相对较低 (1260.16)
- power_age_ratio排名第2，衍生特征价值充分体现
- 更细致的分群

#### 🥈 备选K5的理由
- R²也很高 (0.9707)
- 聚类标签重要性更高 (2.12%)
- 簇数少，便于解释和管理
- 计算效率略高

### 5.3 需要优化的方向

1. **降低RMSE**: 
   - 当前1260-1267，目标是接近之前的1131
   - 可能需要调整超参数或尝试其他算法

2. **超参数调优**:
   - 尝试不同的max_depth (6-10)
   - 调整learning_rate (0.01-0.1)
   - 增加n_estimators (1500-3000)

3. **交叉验证**:
   - 实施K-Fold (K=5或10)
   - 获得更稳定的性能评估

4. **模型融合**:
   - LightGBM + XGBoost + CatBoost
   - Stacking或Blending
